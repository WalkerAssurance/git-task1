Test GIT (VCS) практичне
Мета: Перевірити розуміння та практичні навички роботи з Git, зокрема роботу з віддаленими репозиторіями, гілками та вирішення конфліктів.

Час: 30 хвилин

Завдання:

Створіть новий віддалений репозиторій на GitHub.
Зклонуйте цей репозиторій на свій локальний комп'ютер.
В цьому каталозі створіть новий файл (назва та вміст файлу - на ваш вибір). Додайте цей файл до області збереження Git.
Створіть новий коміт, що включає цей файл. Ваше повідомлення про коміт має чітко відображати, що ви зробили в цьому коміті.
Створіть нову гілку в цьому репозиторії та перейдіть на неї.
Зробіть зміни у файлі, який ви створили раніше, та створіть новий коміт з цими змінами в новій гілці.
Внесіть ще декілька змін у файл, роблячи окремий коміт для кожної зміни. Повідомлення кожного коміту має відображати зміни, які були зроблені.
Надішліть цю гілку та її зміни до віддаленого репозиторію командою git push.
Перейдіть до головної гілки та внесіть зміни в той же файл.
Зробіть коміт з цими змінами та надішліть зміни на віддалений репозиторій.
Спробуйте злити головну гілку з вашою новою гілкою. Ви можете отримати повідомлення про конфлікт.
В такому разі вирішіть цей конфлікт, редагуючи файл та вирішуючи розбіжності між двома версіями. Після вирішення конфліктів, збережіть файл.
Додайте цей файл до області збереження Git та створіть новий коміт, який включає вирішення конфліктів.
Надішліть зміни до віддаленого репозиторію.
В браузерній версії гіт на головній гілці створіть новий файл та додайте в нього будь який текст. Збережіть ці зміни та закомітьте щоб вони зьявились у віддаленому репозиторії.
На своєму локальному комп'ютері виконайте git pull в головній гілці, щоб оновити головну гілку з змінами, які були зроблені в новій гілці.
Перевірте, що всі внесені вами зміни присутні та вірні.
Виконайте в терміналі команди git status, git branch, git log, git log --merges. Створіть новий файл з назвою results.txt на основній гілці та додайте в нього всі результати виконання цих команд.
Збережіть файл за закомітьте його.
Зробіть git push для того щоб усі ваші зміни попали у віддалений репозиторій.
Перевірте чи всі ваші зміни відображено у віддаленому репозиторії через браузер.
Скопіюйте посилання на ваш віддалений репозиторій та вставте у відповідь до цієї форми.
Це завдання потребує вміння використовувати команди git clone, git push, git pull, створення та використання гілок, вирішення конфліктів.
